# Windowsファイルシステム制約

TinyWinNFS Server は、Windows上のフォルダをNFS exportとして公開します。
そのため、NFSクライアントがUnix系のファイルシステムとして扱っていても、実体はWindowsファイルシステムの制約を受けます。

## 大文字小文字

通常のWindowsフォルダでは、ファイル名の大文字小文字は保持されますが、比較では区別されません。

例:

```text
File.h
file.h
```

QNXなどのクライアントでは別ファイルとして扱える場合がありますが、通常のWindowsフォルダでは同一ファイル扱いです。
そのため、コピー中に上書き確認が出たり、片方だけが残ったりします。

v1.10.0以降の診断パッケージでは、大文字小文字を無視した相対パス衝突を検出し、`diagnostics/report.txt` に記録します。

## 予約文字と予約名

Windowsでは、以下のような文字をファイル名に使用できません。

```text
< > : " | ? *
```

また、`CON`、`PRN`、`AUX`、`NUL`、`COM1`、`LPT1` などの予約名も通常ファイル名として扱えません。
NFSクライアント側からこれらを作成しようとした場合、Windows側の制約により失敗します。

## ファイル名長

TinyWinNFS ServerはNFSv2/NFSv3の名前長とWindows側の名前長制約の両方を受けます。
NFSv3 `PATHCONF` の `name.max` は `pathconf.name.max` 設定で応答しますが、実際に作成できるかどうかはWindows側のパス長制約にも依存します。

## symlink

`READLINK` はWindows symlinkのリンク先を返します。
`SYMLINK` はWindows側でsymlink作成が許可されている場合のみ、実体のsymlinkとして作成します。

権限不足やファイルシステム非対応の場合、通常ファイルへフォールバックしてリンク内容を書き込むことはしません。

## QNX運用上の注意

QNX 4.25から大量コピーする場合は、以下を推奨します。

- コピー先exportを空にしてからコピーする。
- `File.h` / `file.h` のような大文字小文字のみ異なる名前を事前に整理する。
- コピー後に診断パッケージを作成し、大文字小文字衝突とexport統計を確認する。
- 上書き確認が出た場合は、前回コピーの残骸または大文字小文字衝突を疑う。

TinyWinNFS ServerはQNX 4.25を重要な検証対象としていますが、製品自体はQNX専用ではありません。
正式版に向けたサポート範囲と対象外機能は [support_scope.md](support_scope.md) を参照してください。
