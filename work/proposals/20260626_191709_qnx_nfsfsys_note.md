# QNX NFSfsys startup note

## Context

The QNX 4.25 client process list was checked after the mount attempt failed with:

```text
No such process
```

The process list showed TCP/IP and other filesystem managers, but did not show `NFSfsys`.

## Diagnosis

The Windows NFS server accepted the QNX MOUNT request for `/export`.
Since the error occurs after the MOUNT RPC succeeds and before NFSv2 file operations start, the most likely cause is that the QNX NFS filesystem manager is not running.

This matches known QNX 4 behavior: `mount_nfs` requires `NFSfsys` to be running on the client side.

## Documentation update

Updated:

- `docs/qnx425_mount.md`

Added:

- `ps -ef` check before mounting.
- `NFSfsys &` startup command.
- Note that `No such process` after a successful Windows MOUNT request usually indicates missing `NFSfsys` on QNX.

## Suggested QNX command sequence

```sh
NFSfsys &
mount_nfs 192.168.1.30:/export /mnt
```

If hostname resolution is confirmed:

```sh
mount_nfs fsnfserver:/export /mnt
```
